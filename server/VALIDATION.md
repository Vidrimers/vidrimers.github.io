# Валидация данных API лайков

## Обзор

API лайков включает комплексную валидацию данных для обеспечения корректности и безопасности работы.

## Валидация projectId

### Формат
- **Тип:** Строка
- **Паттерн:** `(pet|layout)-\d+`
- **Примеры валидных ID:** `pet-1`, `pet-2`, `layout-1`, `layout-2`

### Проверки
1. **Непустая строка** - ID не может быть пустым или null
2. **Корректный формат** - должен соответствовать паттерну `(pet|layout)-\d+`
3. **Существование проекта** - ID должен быть в списке валидных проектов

### Валидные ID проектов
```javascript
const VALID_PROJECT_IDS = [
  'pet-1', 'pet-2', 'pet-3',
  'layout-1', 'layout-2'
];
```

## Валидация POST данных

### Поле action
- **Тип:** Строка
- **Обязательное:** Да
- **Валидные значения:** `"add"`, `"remove"`

### Структура запроса
```json
{
  "action": "add" // или "remove"
}
```

## Коды ошибок

### 400 Bad Request
- Некорректный формат ID проекта
- Отсутствует или некорректное поле action
- Некорректные данные в теле запроса

### 404 Not Found
- Проект с указанным ID не существует

### 500 Internal Server Error
- Ошибка базы данных
- Ошибка сервера

## Примеры ответов

### Успешный ответ GET
```json
{
  "projectId": "pet-1",
  "likes": 5,
  "success": true
}
```

### Успешный ответ POST
```json
{
  "projectId": "pet-1",
  "likes": 6,
  "action": "add",
  "success": true
}
```

### Ошибка валидации
```json
{
  "error": "Некорректный формат ID проекта",
  "message": "ID проекта должен иметь формат \"pet-N\" или \"layout-N\", где N - положительное число"
}
```
